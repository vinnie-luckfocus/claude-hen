## 开发规范

### 规则与规划
- **规则目录**：`[规则目录]`，如 `/rules/`
- **规划目录**：`[规划目录]`，如 `/plans/`

### 任务规划流程
1. 开发前 → 查看 `[规划目录]` 下的规划文档
2. 无对应规划 → 先创建规划文档
3. 开发中 → 严格遵循规划，变更优先更新规划
4. 阶段完成 → 立刻更新规划文档
5. 代码提交 → 审核通过后

### 代码审核
- 每完成一个阶段 → 必须进行代码审核
- 审核内容：
  - 代码逻辑（正确性、性能、安全性、规范）
  - 需求完成度
- 审核通过 → 方可进入下一阶段
- 审核不通过 → 修改 → 复审 → 直至通过

### 文档同步（每阶段执行）

**变更传播**：
- 文件变更 → 更新文件头/元信息
- 模块变更 → 更新模块文档
- 架构变更 → 更新根文档 + 变更记录

**检查清单**：
- [ ] 元信息已更新
- [ ] 文档描述与代码一致
- [ ] 模块文档完整
- [ ] 根文档模块结构最新

### 开发流程

```
1. 读文档 → 建立上下文
2. 改代码 → 实现需求
3. 同步文档 → 各层文档一致
4. 构建验证 → 项目构建成功
5. 代码审核 → 通过后提交
```
