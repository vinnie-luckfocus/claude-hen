# 通用项目初始化规范创建计划

## 概述

深入理解当前项目的开发规范，形成一套可复用的、通用的、用于初始化其他项目的规范。

## 背景

当前项目（planck Mini & Air 智能牙刷固件）已经建立了一套成熟的开发规范，核心包括：
- 分形文档（三层文档结构）
- 回环同构（代码变更后必更新文档）
- Codex 代码审查
- 阶段规划文档更新
- 阶段代码提交

## 目标

创建通用规范，使得这些实践可以快速部署到新项目中。

## 核心要素说明

### 1. 分形文档（三层结构）

```
项目根目录
├── CLAUDE.md              ← 第1层：全局地图
├── components/
│   └── xxx/
│       ├── CLAUDE.md      ← 第2层：模块清单
│       └── xxx.c          ← 第3层：文件头
└── rules/
    └── PROJECT_INIT/      ← 规范定义
```

### 2. 回环同构

```
代码变更
    ↓
更新第3层（文件头 History）
    ↓
更新第2层（如成员/接口/依赖变化）
    ↓
更新第1层（如架构/模块变化）
    ↓
构建验证
    ↓
自检同构
```

### 3. Codex 审查

每个 Phase 完成后：
- 代码逻辑审查（正确性、性能、安全性）
- 需求完成度审查
- 审核通过后方可提交

### 4. 规划文档

- 存放在 `/plans/` 目录
- 每个 Phase 更新进度
- 记录决策和变更

### 5. 阶段提交

- 每个 Phase 完成后提交代码
- 遵循 git workflow 规范
- 包含详细的变更说明
